# Java 聊天室项目解析

## 项目概览：Java 聊天室

这是一个**客户端-服务器（Client-Server）架构**的聊天程序。想象一下微信群聊：有一个中心服务器负责转发消息，每个人的手机上装一个客户端 App 连接到服务器。这个项目就是这样一个东西，只不过是用 Java 写的，运行在电脑上。

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  客户端A  │    │  客户端B  │    │  客户端C  │
│ (Client) │    │ (Client) │    │ (Client) │
└─────┬────┘    └─────┬────┘    └─────┬────┘
      │               │               │
      └───────────────┼───────────────┘
                      │    TCP Socket 连接
               ┌──────▼──────┐
               │   服务器      │
               │  (Server)   │
               │  端口 8888   │
               └─────────────┘
```

## 服务器端（Server）— 3 个核心文件
### 1. [Server.java](Server/src/Server.java) — "接线员"

这是服务端的入口，代码非常短。它做的事情就是：

```java
ServerSocket serverSocket = new ServerSocket(8888);  // 打开 8888 号门，等人来敲门
while (true) {
    Socket clientSocket = serverSocket.accept();     // 有人敲门了，接待他
    new Thread(new ClientHandler(clientSocket)).start(); // 给他安排一个专门的"服务生"
}
```

**比喻**：Server 就像一个酒店前台，坐在 8888 号柜台。每来一个客人，就喊一个服务生（ClientHandler）去单独招待他。来了 100 个客人，就启动 100 个线程，互不干扰。

### 2. [ClientHandler.java](Server/src/ClientHandler.java) — "服务生"

这是整个项目的**核心逻辑**所在，代码最长。每个连接进来的客户端都会分配一个 `ClientHandler`。

它的 `run()` 方法是一个死循环，不断读取客户端发来的消息，然后根据消息**类型**做不同处理：

| 消息类型 | 服务器做什么 |
|---------|------------|
| `register` | 检查用户名是否重复 → 密码 SHA-256 哈希 → 存入 `users.dat` → 回复"注册成功" |
| `login` | 检查用户名密码是否正确 → 检查是否已在别处登录 → 加入在线列表 → 广播"XXX 加入聊天室" |
| `chat` | 把这条消息原样**广播**给所有在线的人 |
| `private_chat` | 找到目标用户，**单独**发给他（私聊） |
| `get_online_users` | 把当前在线人员名单发给请求者 |
| `create_group` | 检查群名是否重复 → 创建群组并自动加入创建者 → 回复创建结果 |
| `join_group` | 检查群是否存在/是否已加入 → 加入群组 → 通知群内其他成员 |
| `leave_group` | 检查是否在群内 → 移出群组 → 通知群内其他成员 → 空群自动删除 |
| `list_groups` | 遍历所有群组找出用户所在的群 → 返回群组名列表 |
| `group_chat` | 检查是否在群内 → 向群内所有在线成员广播消息（含自己） |
| `logout` | 清理资源，广播"XXX 离开了聊天室" |

**关键代码片段解读：**

```java
// 广播：给所有在线用户发消息（除了 exclude 指定的那个人）
private void broadcast(Message msg, ClientHandler exclude) {
    synchronized (onlineLock) {           // 加锁，防止多线程同时修改
        for (ClientHandler handler : onlineHandlers.values()) {
            if (handler != exclude) {     // 跳过不需要接收的人
                handler.sendMessage(msg);
            }
        }
    }
}
```

**密码安全**：密码不是明文存储的，而是用 SHA-256 哈希后存到 `users.dat` 文件里。登录时把用户输入的密码哈希后和存储的比，对上了就说明密码正确。

**群聊机制**：群组采用纯内存管理（服务器重启后丢失），每位用户可同时加入多个群组。

```java
// 群组数据（仅内存，不持久化）
private static Map<String, Set<String>> groups;  // 群组名 → 成员用户名集合

// 向群组所有在线成员广播消息
private void broadcastToGroup(String groupName, Message msg, ClientHandler exclude) {
    Set<String> members;
    synchronized (groupLock) {
        members = groups.get(groupName);       // 先获取成员列表
    }
    synchronized (onlineLock) {
        for (String member : members) {
            ClientHandler handler = onlineHandlers.get(member);
            if (handler != null && handler != exclude) {
                handler.sendMessage(msg);      // 在线才发，离线静默跳过
            }
        }
    }
}
```

群组操作都通过独立的 `groupLock` 保护，与 `userLock`/`onlineLock` 分开，避免死锁。

### 3. [UserInfo.java](Server/src/UserInfo.java) — "用户档案"

很简单，就两个字段：
- `passwordHash`：密码的哈希值（存盘用的）
- `clientSocket`：当前连接的 Socket 对象（标记为 `transient`，不存盘，因为每次连接 Socket 都不一样）

## 客户端（Client）— 4 个核心文件

### 1. [Client.java](Client/src/Client.java) — "启动器"

只有 3 行代码：
```java
SwingUtilities.invokeLater(() -> new LoginFrame().setVisible(true));
```
启动登录窗口。`SwingUtilities.invokeLater` 是为了让界面在**事件调度线程（EDT）**上运行，这是 Swing 的线程安全要求。

### 2. [LoginFrame.java](Client/src/LoginFrame.java) — "登录/注册窗口"

一个 Swing 窗口，包含用户名、密码、服务器地址、端口输入框，以及"登录"和"注册"两个按钮。

关键流程：
- **注册**：建立一个**短连接**，发送 `register` 消息，收到回复后就断开
- **登录**：建立一个连接，发送 `login` 消息，如果成功，**把同一个连接传给 ChatFrame**，登录窗口关闭，聊天窗口打开

### 3. [ChatFrame.java](Client/src/ChatFrame.java) — "聊天主窗口"

**UI 布局（`initUI` 方法）**：

```
┌──────────────────────────────────┐
│  聊天室 - 用户名                  │
├─────────────────────┬────────────┤
│                     │ ┌────────┐ │
│   聊天消息显示区      │ │在线用户│ │
│   (JTextArea)       │ ├────────┤ │
│                     │ │ 群组   │ │
│                     │ │ (选项卡) │ │
│                     │ │        │ │
│                     │ │ 用户列表│ │
│                     │ │ 刷新/私聊│ │
│                     │ │        │ │
│                     │ │ 群组列表│ │
│                     │ │ 创建/加入│ │
│                     │ │ /离开   │ │
├─────────────────────┼────────────┤
│   消息输入框          │ [发送]     │
└─────────────────────┴────────────┘
```

右侧面板使用 `JTabbedPane` 分为两个选项卡：
- **"在线用户"**：用户列表 + 刷新用户 + 私聊按钮
- **"群组"**：群组列表 + 创建群组 + 加入群组 + 离开群组按钮

**后台接收线程（`startReceiving` 方法）**：聊天窗口启动后会开一个后台线程，不停地读取服务器发来的消息：

```java
while (running) {
    Message msg = (Message) in.readObject();  // 阻塞等待消息
    switch (msg.getType()) {
        case "chat":    → 显示 "用户名: 消息内容"
        case "private": → 显示 "[私聊] 用户名: 消息内容"
        case "system":  → 显示 "【系统】XXX 加入了聊天室"
        case "online_list":  → 更新右侧在线用户列表
        case "group_message": → 显示 "[群聊-群名] 用户名: 消息内容"
        case "group_list":    → 更新群组选项卡中的群组列表
    }
}
```

> **为什么需要单独的线程？** 因为 `in.readObject()` 在没消息时会卡住（阻塞），如果放到主线程里，整个界面就会卡死，点任何按钮都没反应。

**发送消息（`sendMessage` 方法）**：把用户输入的文本包装成 `Message` 对象，写入输出流发给服务器。

**私聊与群聊**：

私聊支持两种方式：
- 在输入框打 `/private 张三 你好`（命令行式）
- 在右侧列表选中用户 → 点"私聊"按钮 → 弹窗输入内容

群聊支持三种方式：
- 在输入框打 `/group 技术群 大家好`（命令行式）
- 切换到"群组"选项卡 → 双击群组名 → 弹窗输入内容
- 在"群组"选项卡中可创建/加入/离开群组

### 4. [Message.java](Client/src/Message.java) — "消息协议"

这是客户端和服务器之间的**通用语言**。它是一个 `Serializable` 对象（可以序列化成字节流在网络上传输），包含以下字段：

| 字段 | 用途 |
|------|------|
| `type` | 消息类型（login, chat, private_chat, group_chat 等） |
| `username` / `password` | 登录注册用 |
| `content` | 聊天内容（群聊时存放消息正文） |
| `target` | 私聊目标用户 / 群组名 |
| `from` | 发送者（群聊消息中标识发言人） |
| `success` / `messageText` | 服务器返回的结果（群操作反馈用 `messageText`） |
| `users` | 在线用户列表 / 群组名列表 |

> ⚠️ **注意**：`Message.java` 在客户端和服务端各有一份**一模一样的副本**。因为它们是通过 Java 序列化传输的，两边必须有相同的类定义（包括 `serialVersionUID`），否则会反序列化失败。

## 一个完整的"登录→聊天"流程

```
客户端                        服务器
  │                            │
  │──── register ────────────►│  注册（短连接，用完就断）
  │◄─── register_response ────│
  │                            │
  │──── login ────────────────►│  登录（这个连接会被保留）
  │◄─── login_response ───────│
  │◄─── system "X加入聊天室" ──│  服务器广播给其他在线用户
  │◄─── online_list ──────────│  发送在线名单
  │◄─── group_list ───────────│  发送用户的群组列表
  │                            │
  │──── chat "大家好" ────────►│  发送公聊消息
  │◄─── chat "Y: 你好" ───────│  收到别人的消息（服务器转发的）
  │                            │
  │──── private_chat ─────────►│  发送私聊
  │         (target=张三)       │
  │                            │──── private ───► 张三的客户端
  │                            │
  │──── create_group ─────────►│  创建群组
  │         (target=技术群)     │
  │◄─── system "创建成功" ─────│
  │                            │
  │──── join_group ───────────►│  加入群组
  │         (target=技术群)     │
  │◄─── system "你已加入" ─────│
  │                            │──── system "B 加入了群组" ──► 群内其他成员
  │                            │
  │──── group_chat ───────────►│  群聊消息
  │   (target=技术群,           │
  │    content=大家好)          │
  │◄─── group_message ────────│  自己也收到自己的消息
  │                            │──── group_message ──► 群内其他在线成员
  │                            │
  │──── leave_group ──────────►│  离开群组
  │                            │──── system "B 离开了群组" ──► 群内其他成员
  │                            │
  │──── logout ───────────────►│  退出
  │                            │──── system "X离开了" ──► 广播
  X  连接关闭                   X  清理资源
```

## 对初学者的学习建议

1. **先理解 `Message` 类** — 这是两端的"共同语言"，搞清楚每个字段的用途
2. **服务器端看 `ClientHandler.run()` 里的 switch** — 这是整个项目的"调度中心"
3. **客户端看 `ChatFrame.startReceiving()`** — 理解客户端如何接收和处理消息
4. **关键词理解**：
   - `Socket`：网络连接的"管道"，数据在里面流动
   - `ObjectInputStream` / `ObjectOutputStream`：把 Java 对象变成字节流发出去，再变回来
   - `Thread`：独立的执行路径，让程序能"同时"做多件事
   - `synchronized`：锁，防止多个线程同时修改同一份数据导致混乱
   - `transient`：标记某个字段"不参与序列化"
   - `Serializable`：标记一个类可以被序列化

如果想深入理解，可以试着：修改服务器端口 → 运行服务器 → 运行两个客户端 → 互相发消息看看效果。也可以进一步探索群组功能：创建群组 → 加入群组 → 群内聊天 → 离开群组。从"能跑起来"到"理解原理"是学习编程最有效的路径。
